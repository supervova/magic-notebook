# ✅ Magic Notebook — Production Checklist

## 🎯 0. Перед началом (самое важное)

- [ ] финальный сценарий утверждён
- [ ] тексты титров финальные
- [ ] текст для печати финальный
- [ ] реплика Анны выбрана
- [ ] структура сцен не меняется

## 🎬 1. Скринкасты — подготовка

### Среда

- [ ] production build (`pnpm tauri build`)
- [ ] приложение прогрето (запущено 1–2 раза)
- [ ] devtools закрыты
- [ ] логов нет / минимум

### Desktop

- [ ] чистый desktop
- [ ] уведомления выключены
- [ ] Do Not Disturb включён
- [ ] лишние приложения закрыты

### UI

- [ ] только нужные элементы интерфейса
- [ ] нет сырого/временного UI
- [ ] шрифт читаемый (не мелкий)
- [ ] масштаб окна аккуратный (~1200–1400px)

### Контент

- [ ] подготовлена папка:

```
Writing/
  Drafts/
  Ideas/
  article.md
```

- [ ] файл с текстом подготовлен
- [ ] изображение для вставки выбрано
- [ ] таблица заранее продумана

### Demo typing

- [ ] helper подключён
- [ ] hotkeys работают
- [ ] текст печатается корректно
- [ ] темп устраивает

## 🎥 2. Запись скринкастов

### Настройки

- [ ] 60 FPS
- [ ] high resolution (2× если retina)
- [ ] записывается только окно

### Курсор

- [ ] плавное движение
- [ ] нет дёрганий
- [ ] клики не агрессивные

### Дубли

- [ ] Скринкаст 1 — 3–5 дублей
- [ ] Скринкаст 2 — 5+ дублей
- [ ] Скринкаст 3 — по частям (3–4 дубля на блок)

### Проверка после записи

- [ ] нет лагов
- [ ] нет случайных кликов
- [ ] нет UI-глитчей
- [ ] движения выглядят естественно

## 🤖 3. AI-сцены

### Character consistency

- [ ] есть master образ Анны
- [ ] одинаковые:
  - [ ] волосы
  - [ ] одежда
  - [ ] комната
  - [ ] освещение
  - [ ] ноутбук + 🧐

### Генерация

- [ ] Top view — 5–8 дублей
- [ ] Frontal — 4–6 дублей
- [ ] Talking base — 4–6 дублей
- [ ] Lip-sync — 4–6 дублей

### Проверка качества

- [ ] лицо не “плывёт”
- [ ] руки нормальные
- [ ] ноутбук не деформируется
- [ ] нет лишних объектов
- [ ] камера соответствует сцене

### Talking shot

- [ ] жест с пальцем выглядит естественно
- [ ] взгляд в камеру корректный
- [ ] lip-sync совпадает с речью
- [ ] нет “резинового лица”

## ✂️ 4. Монтаж

### Структура

- [ ] сцены расставлены по таймингу
- [ ] логика не ломается

### Ритм

- [ ] нет спешки
- [ ] есть паузы
- [ ] сцены “дышат”

### Переходы

- [ ] только cut
- [ ] fade используется минимально

### Скринкасты

- [ ] кадрированы
- [ ] при необходимости лёгкий zoom (≤108%)
- [ ] нет дрожания

### Цвет

- [ ] скринкасты чуть контрастнее
- [ ] AI сцены тёплые
- [ ] всё выглядит как единый мир

### Титры

- [ ] только английский
- [ ] короткие
- [ ] без лишней анимации

### Звук

- [ ] музыка тихая (~ -20 dB)
- [ ] нет резких скачков громкости
- [ ] (опционально) subtle UI sounds

## 📦 5. Экспорт

### Master

- [ ] высокое качество
- [ ] 4K (желательно)
- [ ] без агрессивного сжатия

### Web

- [ ] 1080p
- [ ] 8–12 Mbps
- [ ] mp4 + webm

### Субтитры

- [ ] `subtitles.en.vtt`
- [ ] `subtitles.ru.vtt`
- [ ] синхронизация проверена

## 🌐 6. Интеграция в сайт

- [ ] HTML `<video>`
- [ ] `<track>` для субтитров
- [ ] autoplay muted loop (если нужно)
- [ ] fallback для браузеров

## 🔥 Финальная проверка

### Смотри как пользователь

- [ ] понятно без звука
- [ ] хочется досмотреть
- [ ] нет ощущения “рекламы”
- [ ] есть ощущение “хочу попробовать”

### Критерий DONE

👉 если после просмотра возникает мысль:

> “Окей… это просто. Я бы попробовал.”
