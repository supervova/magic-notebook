# Как записать скринкасты (production pipeline)

## 🎯 Цель

Получить:

- идеально плавное видео
- без лагов Tauri/WebView
- с контролируемым курсором
- пригодное для монтажа

## 1. Инструменты

### Запись

👉 **Screen Studio** (лучший вариант)
или
👉 **CleanShot X** (fallback)

Почему:

- стабилизация курсора
- автозум (можно отключить)
- чистый экспорт

### Альтернатива (бесплатно)

👉 OBS + потом обработка (но хуже по UX)

## 2. Подготовка окружения

### 2.1. Подготовь систему

Перед записью:

```bash
## Закрыть всё лишнее
## отключить уведомления
## включить Do Not Disturb
```

### 2.2. Очистка UI

- чистый Desktop
- скрыть:
  - Dock (или оставить минималистичный)
  - лишние иконки

- один workspace

### 2.3. Масштаб

👉 Важно для чёткости:

- системный scaling:
  - **"More Space"** (macOS)

- приложение:
  - не fullscreen
  - аккуратное окно (~1200–1400px ширина)

## 3. Настройки записи

### Screen Studio

- Resolution: **Retina (записывать в 2×)**
- FPS: **60**
- Cursor:
  - size: 110–120%
  - click highlight: subtle

- Background:
  - без фона (чистый захват окна)

### Кадрирование

👉 Записывай:

- либо **только окно**
- либо фиксированную область

❌ не весь экран (лишний шум)

## 4. Подготовка приложения (важно для Tauri)

Tauri может лагать → надо стабилизировать

### 4.1. Production build

❗ НЕ записывай dev-режим

```bash
pnpm tauri build
```

Запускай собранное приложение

### 4.2. Отключить лишнее

Перед записью:

- логи → минимум
- dev tools → закрыты
- heavy эффекты → off (если есть)

### 4.3. Прогрев

Перед записью:

- открыть приложение 1–2 раза
- открыть папку
- прокликать UI

👉 чтобы избежать cold-start лагов

## 5. Курсор (критично)

### Поведение

- движения: плавные, с замедлением
- не “телепортируется”
- не дёргается

### Практика

👉 тренировка 5 минут перед записью:

- клик → пауза → действие
- не водить курсор хаотично

### Хак (очень полезный)

Если сложно:

👉 записывать медленнее
→ потом ускорить x1.1–1.2 в монтаже

## 6. Запись сцен (методика)

### Главное правило

каждая сцена пишется отдельно

### Пример:

#### Скринкаст 1:

- запись 3–5 дублей

#### Скринкаст 2:

- минимум 5 дублей (печать сложная)

#### Скринкаст 3:

- по частям:
  - форматирование
  - изображение
  - таблица

### Почему:

- легче контролировать
- меньше брака
- проще монтаж

## 7. Печать текста (самое сложное)

### Способ 1 (лучший)

👉 печатать вручную

НО:

- тренируешь текст заранее
- знаешь, где паузы

### Способ 2 (если нужно идеально)

👉 использовать вставку + имитацию:

- вставить текст
- удалять/дописывать частями

### Способ 3 (pro)

👉 скрипт печати

## 8. Экспорт из записи

### Настройки:

- формат: **ProRes / high bitrate mp4**
- fps: 60
- без компрессии (или минимальная)

👉 это master-файлы

## 9. Частые ошибки (избегай)

❌ записывать dev build
❌ дерганый курсор
❌ слишком быстрые действия
❌ слишком много движений
❌ грязный desktop
❌ UI с багами/глитчами

## 10. Мини-чеклист перед записью

- [ ] production build
- [ ] папка с файлами готова
- [ ] текст заучен
- [ ] уведомления выключены
- [ ] курсор протестирован
- [ ] 1 пробный дубль сделан
