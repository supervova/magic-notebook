# Детализация скринкастов

## Общие правила (критично)

Перед сценами:

- **FPS:** 60 (потом можно экспортить в 30)
- **Разрешение:** 1920×1080 (или 2560×1440 → даунскейл)
- **Курсор:**
  - без дерганий
  - скорость ~70–80% от обычной

- **UI:**
  - чистый desktop (без мусора)
  - системная тема → светлая (или тёплая)

- **шрифт в редакторе:**
  - читаемый, medium размер (не мелко!)

## 🎬 Скринкаст 1 — Запуск + выбор папки (10–18 сек)

### Цель

Показать: **никакой регистрации / базы / облаков**

### Последовательность

#### 1. Запуск приложения (2 сек)

- клик по иконке в dock
- плавное появление окна

👉 Важно:

- не показывать системные лаги
- окно уже в нормальном размере

#### 2. Empty state (2–3 сек)

Сцена:

- пустое состояние (если есть)

Если нет — можно сразу диалог.

#### 3. Выбор папки (5–6 сек)

Сцена:

- открывается системный диалог
- выбирается папка типа:

```
Writing/
  Drafts/
  Ideas/
  article.md
```

👉 Важно:

- не лазить долго
- 1–2 движения максимум
- уверенный выбор

#### 4. Появление файлов (2–3 сек)

- sidebar заполняется
- клик по `article.md`

### Титр

> Just open a folder.

### UX-нюанс (важно)

Если возможно:

- при первом запуске → автофокус на выборе папки
  это усилит сцену

## 🎬 Скринкаст 2 — Написание текста (18–30 сек)

### Цель

Показать:
**чистоту, поток, отсутствие трения**

### Сцена

#### 1. Новый файл (2 сек)

- кнопка `New note` / `+`
- файл создаётся

#### 2. Печать текста (8–10 сек)

❗ Это САМАЯ важная часть ролика

#### Текст (готовый, используй его):

```
The room is quiet.

There is no pressure to structure ideas,
no need to think about formatting.

Just words, appearing one by one.

And for the first time in a while,
writing feels simple again.
```

#### Как печатать

- скорость: **чуть медленнее обычной**
- паузы:
  - после каждой строки
  - микро-пауза после точки

👉 Это создаёт ощущение «мысли»

#### 3. Минимальные движения

- один раз переместить курсор
- возможно удалить слово и переписать

👉 эффект “живого письма”

### Титр

> And start writing.

### Важно

❌ не показывать:

- панели форматирования
- всплывающие тулбары
- сложные UI элементы

## 🎬 Скринкаст 3 — Редактирование (30–42 сек)

### Цель

Показать **возможности без перегрузки**

### Структура: 3 микро-сцены

### 3.1 Заголовок + выделение (4 сек)

- выделение первой строки
- превращение в заголовок

```
The room is quiet.
↓
# The room is quiet.
```

- затем выделение фразы → bold

👉 важно:

- без меню, если есть хоткей — использовать его

### 3.2 Изображение (5 сек)

#### Сцена:

- drag & drop изображения прямо в текст

Файл:

- что-то спокойное (ноутбук / кофе / окно)

#### После вставки

- изображение появляется
- текст обтекает / сдвигается

👉 ключевое ощущение:
**“оно просто работает”**

### 3.3 Таблица (5–6 сек)

#### Сцена

- вставка таблицы (через кнопку или `/table`)

Пример:

| Idea      | Status      |
| --------- | ----------- |
| Blog post | Draft       |
| Essay     | In progress |

👉 Важно:

- не редактировать долго
- 1–2 ввода максимум

### 3.4 Навигация (2–3 сек)

- клик по другому файлу в sidebar
- затем обратно

👉 показать:
**файловую модель**

### Титр

> No friction. No clutter.

## 🎯 Ключевые принципы для всех скринкастов

### 1. Никакой «демонстрации функций»

Ты не показываешь:

> "смотрите, тут есть таблицы"

Ты показываешь:

> "всё происходит естественно"

### 2. Минимум действий

Каждая сцена:

- 1 идея
- 1–2 действия

### 3. Чистота интерфейса

Если сейчас UI сырой:
→ лучше временно скрыть элементы, чем показывать шум

### 4. Реалистичность

- опечатка → ок
- пауза → ок
- неидеальность → усиливает доверие
