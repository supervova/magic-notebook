# SEO-стратегия

Сервис проверки практической стилистики – это SEO-машина для привлечения пользователей в Magic Notebook. Но сама по себе он не сработает, вокруг него нужно создавать облако лендинг-страниц, статей в блоге и т.д.

## 1. Блог-посты

Можно делать на примере улучшения текстов, сгенерированных ИИ по актуальным темам общемировой повестки. А также – обучающие материалы серии «практическая стилистика для чайников».

- backlinks,
- информационный трафик.

## 2. SEO-оптимизированные страницы сервиса

- существуют как отдельные URL,
- попадают в Google,
- оптимизированы под конкретный поисковый запрос.

Например:

```txt
/readability-checker
/ru/proverka-chitaemosti-teksta
/tautology-checker
/stop-word-checker
```

Это НЕ блог-посты.

### Разница с блогом

Блог-пост отвечает на информационный запрос:

```txt
Как улучшить читаемость текста
Что такое канцелярит
Почему тавтология ухудшает текст
```

Индексируемая tool-page отвечает на намерение/intent:

```txt
readability checker
проверка тавтологии онлайн
канцелярит проверить
stop words checker
```

Это – разные SEO-кластеры.

## 3. Статический рендеринг

В контексте Astro это значит, что HTML страницы генерируется заранее при build Не:

- React SPA,
- client-side rendering,
- «пустой div + JS».

А готовая HTML-страница. Это важно, потому что Google любит:

- быстрые страницы,
- готовый HTML,
- нормальные title/meta/h1/text.

Astro здесь очень хорош.

### Пример

👎 **Плохо** для SEO – `site.com/checker`, где:

- пустая страница,
- всё рисуется JS,
- нет контента без hydration.

👍 **Хорошо** – `site.com/readability-checker`, где уже собраны:

- title,
- meta description,
- h1,
- explanatory text,
- FAQ,
- examples,
- CTA.

А сам checker потом гидратируется как island.

## 3. Будущие страницы

Это НЕ просто разные заголовки одной страницы. Это другая SEO-архитектура.

**Начнём** мы с `/text-checker` – одной страницы со всеми проверками.

**Позже добавим** отдельные landing pages под разные search intents.

```txt
/readability-checker
/tautology-checker
/stop-word-checker
/bureaucracy-checker
/passive-voice-checker
```

Технически это может быть:

- один и тот же React-компонент,
- один и тот же движок,
- одна и та же форма.

Меняются:

- SEO-текст,
- title,
- examples,
- FAQ,
- initial mode,
- keywords,
- hero section.

Это важно, потому что Google ранжирует:

- не «приложение вообще»,
- а страницы под конкретный intent.

Запросы `readability checker` и `tautology checker` — это разные SERP.

### Что это даёт?

Один движок → много SEO-страниц → много long-tail трафика. Так делают:

- Grammarly,
- Hemingway,
- QuillBot,
- Ahrefs,
- Semrush,
- Notion templates galleries.

А для MN это особенно важно, потому что:

- нет рекламного бюджета,
- нет sales,
- нет distribution.

Значит SEO — один из немногих реальных каналов роста.
